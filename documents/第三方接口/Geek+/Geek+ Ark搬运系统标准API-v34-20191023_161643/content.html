接口规范说明

协议说明

主机系统可以使用TCP SOCKET、HTTP协议建立与RMS之间的连接。这三种协议同时只能使用一种。

协议端口号备注
TCP Socket24245
HTTP24249

必须使用 HTTP 请求中的 POST 方式；

 HTTP 头中的 MIME 类型需要配置为“application/json”

格式说明

  1. 格式一律采用json形式；
  2. 报文以/r/n结尾,报文中不能有空格和空行；

消息类别

根据消息功能不同，将所有消息分为了4类，如下表所示。

消息类别

功能描述

消息发起者

消息接收者

请求消息

主机系统向极智嘉搬运系统发送业务指令：领单/料，完成，取消等等主机系统极智嘉搬运系统
请求响应消息是指上一类请求消息发送后，极智嘉搬运系统做出的返回结果极智嘉搬运系统主机系统
回调消息极智嘉搬运系统在任务执行的阶段进行的结果返回极智嘉搬运系统主机系统
回调响应消息是指上一类请求消息发送后，主机系统接收回调后的响应主机系统极智嘉搬运系统

请求消息

请求消息详细说明见后续章节的接口描述


请求响应统一如下表：

参数列表字段名称数据类型长度是否必填版本字段含义备注
responseId
VARCHAR32Y

包含在消息头中，当前回调应答消息的唯一标识。

用于保证消息的幂等性，当前响应消息与对应的回调消息的requestId保持一致。


code
VARCHAR32Y
包含在消息头中，回调应答代码，成功：0，失败：非0。
